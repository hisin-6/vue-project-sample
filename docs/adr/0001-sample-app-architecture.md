# ADR 0001: サンプルアプリの基本方針

- 日付: 2026-02-05
- ステータス: 提案

## コンテキスト
サンプルアプリの初期方針を整理して記録する。現時点で確定している要件は以下。

- フロントエンドはVueを使う
- 最終的にEC2へデプロイする
- アプリで入力した内容をAWS S3へCSV出力する

## 決定
以下を基本方針として採用する。

1. UI実装はVueを採用する
2. デプロイ先はEC2を前提に設計する
3. 入力データの出力先はS3、形式はCSVとする
4. ローカルでの確認時はS3の代替としてMinIOを利用する

## 根拠
- Vueは学習コストと開発効率のバランスがよい
- EC2は柔軟な運用が可能で、段階的な拡張に対応しやすい
- S3はCSVの保存・取得・運用が容易

## 影響・トレードオフ
- EC2運用のためインフラ管理が必要
- CSV出力のため、スキーマ変更時に互換性対応が発生する可能性

## 今後の検討事項
- バックエンドの実装方式（API、バッチ、認証）
- CSVのスキーマとバリデーション
- S3への書き込み方法（直接アップロード or サーバー経由）
